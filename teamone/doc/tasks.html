Tasks #

Tasks allow you to create automations for different TM1 routines (running TI processes, import and export TM1 data, etc.). Using TeamOne > Configuration menu can quickly add new tasks templates to TEAMONE configuration sheet and then modify them as needed. You can use Sheets formulas to make tasks dynamic. A common example would be to use =YEAR(TODAY()) formula to get the current calendar year and use it as a parameter for TM1 process task. All the defined tasks can be run from the TeamOne sidebar:

Each task is defined by specifying a ::tasktype in column A and task name in column B. To execute a task you simply click it in TeamOne sidebar.

Adding TM1 tasks #

Tasks are one-click automations that can be executed from TeamOne sidebar. They include: tm1process, tm1view, tm1import and tm1fileoutput (more automations are coming in the future). tm1process #

tm1process task is used to run TI processes. See the table below how to specify the process name and parameters:

Option / parameterDescription
-nameUsed to provide a TI process name. If omitted the task name will be used
-tm1envindicates to run the process always in the specified environment. If omitted the process will run in the currently selected environment
-infoA task hint, will be displayed when hovering a mouse cursor over info icon next to the task name
parameter 1-NYou can provide as many process parameters as needed

To execute a task you simply click it in TeamOne sidebar. When the process is completed you will see a pop up message confirming the execution status.

tm1view #

tm1view task allows to query TM1 data (by specifying an MDX expression) and print it on a sheet.

Option / parameterDescription
-tm1envIndicates to extract data in the specified environment. If omitted the currently selected environment will be used
-infoA task hint, will be displayed when hovering a mouse cursor over info icon next to the task name
sheetSheet name to output the data to
mdxMDX expression to query TM1 data
activateIf provided, it will switch to the specified sheet automatically once the data is printed

tm1import #

tm1import task is used to import data from sheets into TM1 cube:

Option / parameterDescription
-tm1envIndicates the environment to use. If omitted the currently selected environment will be used
-infoA task hint, will be displayed when hovering a mouse cursor over info icon next to the task name
cubeTM1 cube name to import the data into
sheetSheet name containing the data
startRowIndicates the first data row. If omitted the default value 1 will be used
startColumnIndicates the first data column. If omitted the default value 1 will be used

There are several data requirements:

  1. The column order should match the cube dimensions
  2. A separate column should be used for each measure dimension element
  3. Measure dimension elements should match the principal dimension element names in TM1
  4. The data types should match the dat types of the measure dimension elements

tm1fileoutput #

tm1fileoutput task allows to export sheet data into a field delimited file on TM1 server.

Option / parameterDescription
-tm1envIndicates the environment to use. If omitted the currently selected environment will be used
-infoA task hint, will be displayed when hovering a mouse cursor over info icon next to the task name
sheetSheet name containing the data
startRowIndicates the first data row. If omitted the default value 1 will be used
startColumnIndicates the first data column. If omitted the default value 1 will be used
rowsNumber of rows to export. If ommited or 0, all data rows will be used
columnsNumber of columns to export. If ommited or 0, all data columns will be used
fileFile path on TM1 server. You can specify a relative path to the TM1 data directory, for instance ..\FileName.txt will save FileName.txt on level folder up
delimiterfile field delimiter, if omitted a tab character will be used
