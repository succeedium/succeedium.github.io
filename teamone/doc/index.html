Getting started ​

Get TeamOne ​

TeamOne can be easily added to Google Sheets™ from Google Workspace Marketplace™ (registered trademark owned by Google LLC ) by following this link. Google Workspace™ subscription is not required to use TeamOne, as it can work with a free account. However, if you have a paid plan, you can easily install and configure TeamOne for all your users at once using the domain settings.

SSL certificate ​

IBM PA Local only. We highly recommend configuring a custom SSL certificate (signed by a trusted certificate authority) to prevent any browser warnings and ensure that your connection is secure. Please refer to the IBM Planning Analytics documentation for detailed instructions on how to configure SSL for your IBM Planning Analytics server version. You can start with this link: How to Configure SSL on IBM Planning Analytics.

TeamOne menu ​

You can find TeamOne under the Extensions menu. It includes options to open the TeamOne Sidebar, TeamOne Configuration, TeamOne Getting started, access additional tools and see the About page which displays the current add-on version and your license information.

TeamOne configuration ​

You can manage TeamOne add-on settings either through the TeamOne Configuration or on a sheet named TEAMONE. With the TeamOne Configuration, you can define personal, document, and domain connections and tasks. Meanwhile, the TEAMONE sheet allows you to edit the settings and even make them dynamic by using sheets formulas.

Settings hierarchy ​

It's possible to define TeamOne settings at four different levels, as follows:

  1. Domain Settings - available to all corporate users across all Google Sheets documents.
  2. Document Settings - available to all corporate users in a particular Google Sheets document.
  3. Personal Settings - available to a specific user in all Google Sheets documents.
  4. TEAMONE Sheet - defined in a specific Google Sheets document on a tab named "TEAMONE."

Personal settings ​

In My Settings, you can define private IBM PA connections and private tasks. These personal settings are stored in your Google account and shared across all your Google Sheets documents where you use the TeamOne add-on. It is important to note that no other users can access or view these settings.

Document settings ​

Document Settings are saved in the current Google Sheets document properties and are accessible to all users of a given document. However, only Admin users can view and modify these settings.

Domain settings ​

Domain Settings are available to all corporate accounts. However, only designated domain administrators can view and modify these settings.

Additionally, you can specify the permissions and list the users who can use each IBM PA connection.

TEAMONE sheet ​

You can define IBM Planning Analytics connections and tasks directly on a sheet named TEAMONE. You can use Sheets formulas, copy-paste settings, or comment out individual settings or entire settings blocks.

Merging settings ​

If you have the same named Personal, Document, Domain and TEAMONE sheet settings, they will be merged in the next order:

  1. Domain Settings
  2. Document Settings
  3. Personal Settings
  4. TEAMONE sheet

For instance, as an admin user, I would like to define a new IBM PA connection named IBM Cloud PROD in the Domain or Document Settings, this is where I would define the server host, port and namespace (in case of CAM authentication). Then other users can also define IBM Cloud PROD connection in their Personal Settings, but populate the username/password fields only. Finally on TEAMONE sheet we can aslo define IBM Cloud PROD connection and specify a value for the securitySheet. When opening TeamOne Sidebar, all the settings will be merged, so IBM Cloud PROD connection will get the host, port and namespace from the Domain or Document Settings, the secret from the Private Settings, and finally the securitySheet value defined on the TEAMONE sheet.

Security ​

When you start using the TeamOne add-on in a new Google Sheets document, no security is defined by default. This means that all users can access the Document Settings and modify the Security. To set up security, go to the Security tab and add the email addresses of users who should be able to use TeamOne in a given document. Once you add your first account, other users will not be able to edit the Document Settings and Security tabs, unless they are added on the Security tab with admin permissions.

TIP

Use Google Sheets sharing options to specify who can access your document and who can edit it. You can also use Protected sheets and ranges to control who can modify the TEAMONE config sheet.

IBM PA connection ​

You will need at least one IBM PA connection, which can be defined in either the TeamOne configuration or in TEAMONE sheet. To establish a connection to the TM1 server, you will need to provide specific parameters depending on whether you are using an IBM PA Cloud or IBM PA Local connection. These parameters include:

  • -name: This parameter allows you to assign a name to the connection, which can be different from the actual IBM PA server name. - * -host: This parameter requires you to specify the IBM PA server host FQDN. Please note that you should not specify the protocol as TeamOne will always use https://.
  • -port: This setting should be equal to HTTPPortNumber in the tm1s.cfg file. You can omit this parameter if using 443 (default for HTTPS).
  • -server: (IBM Planning Analytics Cloud only) Specify your database name, case sensitive!
  • -proxy: (IBM Planning Analytics Cloud only) Available to the clients who purchased SPACE server, used to enable IBM SSO and improve the PA Cloud connection speed. Leave it empty otherwise.
  • if using IntegratedSecurityMode 4 or 5 specify the -namespace configured in Cognos Analytics (usually "LDAP"). Use "LDAP" if using IBM Planning Analytics Cloud connections with non-interactive user account. Not needed if using -proxy and IBM SSO.
  • When creating or updating IBM PA connection in TeamOne configuration, you can specify your -username and password, TeamOne will automatically generate and store the corresponding secret.

TIP

When you connect to the IBM PA server, TeamOne will verify if your user account belongs to the ADMIN security group. If your account has the ADMIN security group, you will be able to access additional functionalities in the sidebar. These functionalities include tabs for Users, Logs, and Processes, as well as menu options for TM1 console, TM1top, and Compare models.

IBM PA Cloud connection ​

To connect to IBM Planning Analytics Cloud, you will need the following information:

  • The -host setting should be the same as your IBM Planning Analytics Workspace URL, which usually looks like: YourCompany .planning-analytics.ibmcloud.com.
  • The -server setting refers to the database server name that you can find in the PAW database list. Please note that the server name is case sensitive.

There are two ways to authenticate to IBM PA Cloud: using a non-interactive account or by configuring IBM Single-Sign-On (SSO).

Non-interactive accounts ​

Non-interactive accounts are special type of accounts that allow applications to connect to IBM PA Cloud. However, it is important to note that these accounts cannot be used to log into IBM Planning Analytics Workspace.

To connect to IBM PA Cloud using a non-interactive account, you will need to set the "-namespace" connection parameter to LDAP. You will also need to use the credentials of the non-interactive user account provided in the IBM PA welcome kit, which includes the necessary information to establish the connection.

Using IBM SSO ​

Using Single-Sign-On (SSO) you will be able to connect to IBM PA with your email & password saving the credentials in TeamOne, as shown in this video.

To begin, you will need to create a support ticket with IBM to obtain your CLIENT_ID and CLIENT_SECRET. Once you have received these credentials, proceed to set them up in the configuration settings of your SPACE Server, which comes bundled with your purchase of TeamOne.

If you need help with either of these steps, please contact us at teamone@succeedium.com and we'll be happy to assist you.

SPACE Server

By using the SPACE server, you can greatly enhance the connection speed between TeamOne and IBM Cloud. Additionally, it enables IBM SSO functionality, which allows you to log in to IBM PA through Google Sheets using your email and password. Moreover, SPACE seamlessly integrates Python and a plethora of other features directly into Planning Analytics Workspace. You can learn more about it by visiting https://succeedium.com/space/.

TeamOne Sidebar ​

You can access TeamOne Sidebar from the Extensions menu. If you have no TM1 connection defined, TeamOne Configuration screen will pop up automatically where you can add one.

The sidebar toolbar allows you to select the current environment, refresh the config (if you made any changes on TEAMONE sheet or in Configuration menu) and do several quick actions from the main menu:

  • Search TM1 model
  • Compare models
  • Update security, if you defined a server secruity sheet
  • Save data on TM1 server
  • Open TI console
  • Open TM1 Top
  • Open TeamOne Configuration

Refreshing configuration ​

When you open the TeamOne sidebar, the configuration is automatically reloaded. Use the Refresh Config button after making any changes to the TEAMONE sheet or through TeamOne Configuration.
